3. Largest prime factor

Read the problem definition at projecteuler.net

The prime factors of 13195 are 5, 7, 13 and 29.

What is the largest prime factor of the number 600851475143 ?

This challenge is much more difficult than the previous two, where the most obvious brute force solutions were optimal. If we try a simple brute force solution here, such as checking every natural number less than our given number for primeness and divisibility into the given number, our code will take a long time to execute. If we have lots of computing power or time it's still feasible to brute force it that way. However, there's a better algorithm.

The technique we're going to use depends on the fundamental theorem of arithmetic, which states that all natural numbers greater than 1 are the product of a unique set of prime factors.

This algorithm divides the input by the smallest valid factor, starting with 2 and incrementing by 1 until it finds a number that divides in evenly. It then recurses the function using the quotient from that division as the new input. This process guarantees that each divisor will be prime, as all numbers smaller than it (those which could be factors) will have already been tried as factors for the original input and failed.

Once the function runs on an input that has no factors the algorithm stops, having found the last and largest member of the set of unique primes whose product is the original input.

The code here correctly identifies 6857 as the largest prime factor of 600851475143.
