4. Largest palindrome product

Read the problem definition at projecteuler.net

A palindromic number reads the same both ways. The largest palindrome made from the product of two 2-digit numbers is 9009 = 91 Ã— 99.

Find the largest palindrome made from the product of two 3-digit numbers.

This problem deals with numeric palindromes: numbers whose digits form the same sequence whether read forward or backward, like 123321.

Solving this challenge requires comparing individual digits of large numbers. To do this, we'll transform the numbers into strings with the toString() method. We can then access individual digits like array indices with the charAt() method.

An important task in this algorithm is determining if a number is palindromic. We'll write another function called checkIfPalindrome() to handle this. It converts the input it's given into a string and sets two indices, one at the beginning of the string and one at the end. It then checks to see if the characters at those indices match, and returns false if they don't. If the first and last characters match, each index is moved one character towards the middle of the string and the loop repeats. Characters equidistant from the middle of the string are checked in this manner until the middle of the string is reached, indicating a palindromic input.

Our main function uses two nested loops to produce products of 3-digit numbers. We choose not to check the products of numbers less than 500, as it seems reasonable to conclude that the largest palindrome will be a product of two larger-than-average 3-digit numbers. Each product we produce that's bigger than the biggest palindrome found so far is checked to see if it's palindromic. If so, it gets stored as the largest palindrome found.

When the loops complete, we're left with 906609 as the largest palindrome found, which is the correct answer to the challenge.
